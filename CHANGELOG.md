## [1.0.0-beta12] - 2026-06-20

本次为 Windows 端大版本更新，汇总 `v1.0.0-beta7` 之后的主要改动，重点补齐图生图、局部重绘、扩图、本地工作流、Krita 联动、Vibe、提示词编辑和生成稳定性。

### Added

- **完整图生图与局部重绘工作流**：补齐图生图、局部重绘、Focused Inpaint、Outpaint 边缘扩展、点击式填充桶和虚拟画布扩图流程。
- **ComfyUI 本地工作流**：新增本地 ComfyUI 接入、工作流导入、内置超分和相关设置入口。
- **Krita Bridge 联动**：新增 Launcher 与 Krita 的本地桥接工作流，支持从生成结果、历史记录和图库发送图片到 Krita。
- **提示词 Token 明细**：正向/负向提示词支持 Token 计数、组成明细和紧凑显示。
- **提示词框内搜索**：长提示词现在可以直接在编辑器内搜索和定位。
- **Vibe 导入导出增强**：支持非 PNG Vibe 导入、Vibe 整包导入参数保留、PNG 元数据嵌入导出和批量导出。
- **精准参考增强**：Precise Reference 支持多图导入，拖放角色参考图时会追加到已有参考列表。
- **随机配置模式**：支持官方、自定义、混合模式持久化，并兼容旧随机预设。
- **本地图库多选增强**：本地图库支持当前页/全部结果两种多选范围，历史记录增加更多图片目标快捷操作。
- **反推入口补充**：反推可以作为全局拖图目标，并打通更多图片发送入口。

### Changed

- **生成取消逻辑更清晰**：连续生成时区分“跳过当前批次”和“停止全部”，避免误以为全部任务已停止。
- **批量生成更稳定**：重复生成和多样本批次分开处理，减少进度、取消和结果回写混乱。
- **流式生成失败更好排查**：生成失败或取消时会保留最后预览快照和元数据，方便进入详情查看。
- **图像编辑体验更一致**：局部重绘、扩图、硬边蒙版导出和预览图操作进一步对齐实际发送结果。
- **元数据兼容性提升**：保存、读取、导入和复用图片参数时更贴近 NovelAI 原生格式。
- **自动补全更贴合 NAI 语法**：自动补全会保留提示词外层语法包裹，减少权重或括号结构被破坏。
- **提示词助手更稳**：图片输入会在发送前自动压缩和规范化，降低请求失败概率。
- **图库与大图预览性能优化**：优化缩略图、缓存、Vibe 库导入和大图处理，减少卡顿。
- **Windows 文件定位更可靠**：优化“在文件夹中显示”相关操作，减少路径打开失败。
- **源码构建环境更新**：最低要求调整为 Flutter 3.35.0 / Dart 3.10.7。

### Fixed

- 修复生成取消链路中的多类问题，包括旧结果回写、空结果、取消后恢复异常和批量取消状态不明确。
- 修复局部重绘蒙版偏移、丢边、预览与发送不一致、扩图 frame 校验等问题。
- 修复 NovelAI 超分、变体、Focused Inpaint 等场景下的 Anlas 估算偏差。
- 修复本地图库逗号搜索、分类删除、图片元数据覆盖和部分搜索状态不同步问题。
- 修复 Vibe 库周期性卡顿、Vibe PNG 元数据回写和导出流程本地化缺失。
- 修复词库搜索框内全选快捷键被页面快捷键抢走的问题。
- 修复固定词侧栏中容易误触的权重快捷操作。
- 修复英文界面中部分中文文本泄漏的问题。
- 修复 ComfyUI 地址、模型解析、工作流导入和执行中的多处兼容问题。
- 修复 Krita Bridge 预检、安装、发送图片和局部重绘返回中的稳定性问题。

### Contributors

## [1.0.0-beta11] - 2026-06-13

本次更新基于 PR #8-#16 合并整理。

### Feature

1. 新增 Krita Bridge 本地桥接工作流，支持 Launcher 侧认证 WebSocket 服务、Krita 插件、设置页开关、生成/图库发送到 Krita、插件安装/打包/预检和验收脚本。
2. 完善图像编辑与重绘/扩图流程，新增 Outpaint 边缘扩展、虚拟画布扩图状态、导出时物化扩图源、硬边蒙版导出器和 CPU/Isolate 处理路径。
3. 增强参考图与图片输入能力，精准参考支持多图导入，生成页支持从本地拖放图片到 Vibe Transfer 与 Precise Reference。
4. 完成随机配置模式，支持官方/自定义/混合模式持久化、随机预设规范路由和旧预设兼容。
5. 提示词助手会在发送前自动压缩过大的图片输入，并保持刷新过程中的 Token 统计可见。
6. Vibe 整包导入会保留子项参数与原始图片来源，便于后续详情查看、编辑和生成复用。
7. 本地图库增加当前页/全部结果两种多选范围，历史记录增加图片目标快捷操作，反推也可作为全局拖图目标。
8. 生成过程中桌面端取消按钮始终可见，并在批量生成时显示进度；流式生成失败或取消时会保留最后预览快照和元数据，便于进详情排查。

### Bugfix

1. 修复生成取消链路中的多类竞态，包括取消后旧结果回写、空 fallback 结果、预览前未取消 stream、批量取消失败未暴露，以及取消中的请求恢复问题。
2. 修复 NovelAI inpaint 蒙版合成与预览图操作一致性，避免局部重绘蒙版在发送、预览或导出时偏移、丢边或被错误合成。
3. 修复图像编辑器虚拟扩图与硬边蒙版导出问题，包括 frame snapping、物化 frame 校验、图层 offset、基础蒙版合成和 DTO 字节可变性。
4. 修复本地图库逗号搜索筛选、在线图库搜索和自动补全细节，并优化大批量图像处理与 mask region fill 性能。
5. 修复 Vibe 库导入确认卡顿、Vibe PNG 元数据回写、导出流程中文本地化缺失，以及紧凑操作栏溢出问题。
6. 修复可见中文字符串在英文界面泄漏的问题，并补齐设置、Vibe、队列、快捷键、拖放、详情页等多个界面的本地化文本。
7. 修复 Krita Bridge 预检包输出路径、局部重绘返回速度和释放安全性问题，减少外部工作流联动中的失败与卡顿。
8. 修复元数据导入中 disabled Variety+ 状态、图片元数据覆盖、文件夹显示入口和若干生命周期回归，并增加吞掉异常的日志记录以便排查。

## [1.0.0-beta8] - 2026-05-02

### Added

- **词库中文搜索反查** - 中文搜索词现在可以通过翻译数据库定位对应英文 Danbooru tag，并保留分类与热度排序信息。
- **完整图生图与重绘工作流** - 补齐 `img2img`、局部重绘、变体、增强等生成流程，并引入 `Focused Inpaint`、`Minimum Context Area` 与点击式填充桶。
- **可持续编辑的重绘画布** - 重绘编辑器支持再次进入后继续查看和修改已有蒙版，Focused 双框可视化与主蒙版编辑逻辑统一。
- **ComfyUI 本地工作流与内置超分** - 新增 ComfyUI 集成、工作流导入、设置页与内置 `SeedVR2` 超分，可直接在应用内切换本地超分后端。
- **导演工具独立工作流** - 新增导演工具页面与入口，支持从历史图、预览图和图生图源图直接送入增强处理。
- **提示词 Token 计数与明细** - 为正向和负向提示词增加 Token 计数、组成明细与紧凑模式显示，并纳入固定词、质量预设、UC 预设和角色提示词。
- **复制/拖拽元数据控制** - 新增“复制/拖拽时移除全部元数据”设置，同时保留本地保存图片的完整元数据。
- **Vibe 导入导出增强** - 支持非 PNG 的 Vibe 导入，并新增带 PNG 元数据嵌入的 Vibe 导出流程。
- **反推模块与本地 Tagger 工作流** - 新增“反推”链路入口，支持 ONNX Tagger 接入、结果回填提示词，并打通图库/画布图片送入反推。
- **多任务 LLM 助手路由** - 助手补齐优化、翻译、反推、角色替换任务路由，新增对应规则管理与提示词框注入入口。
- **保护模式细分开关** - 设置页新增保护模式并拆分子开关，覆盖拖拽净化、危险操作确认、外发提醒与高消耗预警。
- **本地 Effects 工作流补充** - 图像编辑区域新增本地效果处理入口，支持更完整的本地后处理操作。

### Changed

- **Vibe 导出流程更清晰** - 单个 Vibe 可直接另存为文件或嵌入 PNG，批量导出改为先选择目录并自动避免文件名覆盖。
- **元数据格式更贴近 NovelAI** - 保存与解析逻辑更贴近 NovelAI 原生风格，网页端拖入兼容性更好，同时应用自身仍能正确回读设置。
- **应用内拖拽与资源管理器拖拽对齐** - 在保留元数据时，应用内拖拽改为优先使用真实落盘文件字节，减少与资源管理器拖拽行为不一致的问题。
- **本地超分参数默认更保守** - 内置 `SeedVR2` 本地超分默认切到 `3B Q4` 模型与 `1.5x` 倍率，并把超分/增强参数改为和常规生成参数一样持久化。
- **历史记录交互更完整** - 历史列表与预览卡片的悬停栏、右键菜单现在都能直接进入局部重绘和放大。
- **SD 语法自动转换职责收敛** - `SD语法自动转换` 现在只负责将 SD 权重语法转换为 NAI 语法，空格转下划线统一交给自动格式化处理。
- **超分后端收口到 ComfyUI** - 移除独立本地 ONNX 放大路径，统一由 ComfyUI 工作流承载 `SeedVR2` 与常规超分模型选择与执行。
- **LLM 请求策略收敛为非流式** - 关闭流式输出路径，移除温度/Top-P/Max Tokens UI 配置并回退云端默认参数，减少截断和空响应问题。
- **提示词编辑撤销重做链路强化** - 注入式改写（助手/反推）统一接入可撤销历史，补齐 `Ctrl+Y` 重做行为。

### Fixed

- **修复词库搜索框状态不同步** - 工具栏重建后会重新同步当前搜索条件，避免输入框显示和实际筛选状态不一致。
- **修复批量 Vibe PNG 嵌入误入口** - 批量导出不再显示仅适用于单个 Vibe 的 PNG 嵌入选项，避免无效操作路径。
- **修复重绘灰块、灰边和蒙版显示异常** - 解决普通重绘与 Focused Inpaint 中的灰块、灰边、整块回贴、蒙版层级错误与清空后继续编辑不可见等旧问题。
- **修复 Vibe 库周期性卡顿** - 收敛 Vibe 库后台刷新、从库导入与缩略图解码路径带来的周期性卡顿与短时冻结。
- **修复生成页稳定性问题** - 解决冷启动切换负面提示词可能闪退、自定义尺寸输入被覆盖、更新检查 race 和缩略图失败重试风暴等旧问题。
- **修复大图预览性能问题** - 历史记录、画廊、精准参考等区域改为更节制的图像解码策略，降低大图场景下的卡顿。
- **修复 NovelAI 与本地超分结果不一致问题** - 解决 NovelAI 超分缺少倍率参数、ComfyUI 模型列表被错误解析成 `COMBO`、放大结果排序和详情分辨率错误的问题。
- **修复 Anlas 估算偏差** - 重新按用户订阅等级、实际请求尺寸和工作流模式计算 `Anlas`，覆盖 NovelAI 超分、变体和 `Focused Inpaint` 的真实消耗。
- **修复 ComfyUI 地址格式导致的假连接失败** - 统一规范化 base URL 与 WebSocket 路径，避免 `//ws` 等异常导致的任务执行失败。
- **修复本地图库送图链路缺失** - 打通本地图库卡片与详情页到 `img2img` 的发送入口，并补齐通用图片操作路径。
- **修复 Vibe 导入阶段全量读取** - 导入流程去除 `getAllEntries()` 重负载路径，改为按需查找与轻量读写。
- **修复 Vibe 展示缓存过重导致的首开卡顿** - 展示缓存升级为轻量 `v2`，剥离大字段并改为按需懒加载缩略图。
- **修复大体积元数据缓存后台清理卡顿** - 为超大 L2 Hive 缓存增加自动清理硬阈值保护，避免后台全量扫描/compact 触发长卡顿。
